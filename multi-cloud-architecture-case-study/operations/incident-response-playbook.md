# Incident Response Playbook – Data Platform

1. **Detect** – Incident raised by alert or user report.
2. **Triage** – Identify affected pipelines, datasets, and users.
3. **Contain** – Disable offending jobs or access paths if security‑related.
4. **Restore** – Re‑run failed pipelines, restore from backups if needed.
5. **Review** – Run a blameless post‑incident review and capture action items.
